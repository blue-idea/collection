## 七维审查结论

| 维度 | 结论 |
|------|------|
| 术语一致性 | “系统 UI 文案”定义为产品提供的可见文本与无障碍名称；“自定义内容”定义为用户输入、导入或 AI 生成的数据内容。 |
| 用户角色清晰度 | 本地用户与登录用户均适用，权限边界不变。 |
| 触发条件完整性 | 语言来源固定为已持久化 `settings.locale`；Settings 草稿仅用于即时预览。 |
| 系统响应可验证性 | 可通过词典键完整性、组件渲染、E2E 可见文本、`aria-label`、`html[lang]` 与截图验证。 |
| 前置条件与约束 | 仅支持 `en` / `zh`；English 为安全回退；用户自定义内容不得被翻译或改写。 |
| 跨文档冲突 | 数据模型和 API 无需变化；既有设计中“部分 CTA 固定英文”已改为 i18n key。 |
| 非功能性需求 | 保留可访问名称、持久化兼容和业务行为；增加硬编码审计与视觉回归门禁。 |
