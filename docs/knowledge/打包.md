当前项目如何正式构建windows、macos 应用

---

## 开发 / 发布身份隔离（重要）

本地用户数据与 AI 密钥**不会**打进安装包；它们写在本机身份槽中。开发与发布必须使用不同槽位，否则在同一台机器上验证 Release 会看到开发期数据：

| 构建 | 命令 | AppData 目录 | Keychain 服务名 |
|------|------|--------------|-----------------|
| 开发 | `scripts/dev.ps1` / `scripts/dev.sh`（`wails dev -tags dev`） | `Linkit-Dev` | `Linkit-Dev` |
| 正式 | `wails build`（CI Release **不加** `dev` tag） | `Linkit` | `Linkit` |

Release 工作流在干净的 GitHub runner 上构建，产物本身不含用户数据。本机验证正式版时若仍看到测试数据，说明曾用正式身份（无 `dev` tag）跑过开发构建并写入了 `Linkit` 槽。

本地快速核对：

```bash
go run ./scripts/check-identity          # profile=release → .../Linkit
go run -tags=dev ./scripts/check-identity -write-probe  # 仅写入 Linkit-Dev
```

项目使用 Wails 框架构建桌面应用，以下是正式构建方式：

## 本地构建

前提条件：Go、Node.js、pnpm、Wails CLI

```bash
# 安装 Wails CLI（版本与 CI 一致）
go install github.com/wailsapp/wails/v2/cmd/wails@v2.13.0

# Windows 构建
wails build -clean -platform windows/amd64
# 产物：build/bin/linkit.exe

# macOS 构建（需在 macOS 机器上执行）
wails build -clean -platform darwin/universal
# 产物：build/bin/Linkit.app
```

Wails 会自动处理前端依赖安装和构建（通过 wails.json 中的 `frontend:install` 和 `frontend:build` 配置）。

## CI 自动构建

项目已配置 GitHub Actions，推送到 `main`/`master` 分支或 PR 时自动触发：

- Windows 构建运行在 `windows-latest` runner
- macOS 构建运行在 `macos-latest` runner（`darwin/universal` 同时支持 Intel 和 Apple Silicon）

工作流文件：[.github/workflows/desktop-build.yml](.github/workflows/desktop-build.yml)

## 发布 Release

通过 GitHub Actions 手动触发 [release.yml](.github/workflows/release.yml)，输入 tag 名称即可创建 GitHub Release（目前 release 工作流构建的是 `linux/amd64`，如需发布 Windows/macOS 产物需要修改该工作流）。

如何构建安装包

---

项目目前只构建出裸可执行文件（`.exe` / `.app`），**没有配置安装包**。以下是两个平台的推荐方案：

---

## Windows — NSIS 安装包

Wails 内置 NSIS 支持，只需加 `-nsis` 参数：

```bash
wails build -clean -platform windows/amd64 -nsis
```

前提：需要先安装 [NSIS](https://nsis.sourceforge.io/Download)，并确保 `makensis` 在 PATH 中。

产物：`build/bin/linkit-amd64-installer.exe`

---

## macOS — .dmg 安装包

Wails 不内置 dmg，需借助 `create-dmg`：

```bash
# 安装工具
brew install create-dmg

# 先构建 .app
wails build -clean -platform darwin/universal

# 打包为 dmg
create-dmg \
  --volname "Linkit" \
  --window-size 600 400 \
  --app-drop-link 400 200 \
  "build/bin/Linkit.dmg" \
  "build/bin/Linkit.app"
```

产物：`build/bin/Linkit.dmg`

> macOS 正式发布还需要 Apple Developer 账号进行代码签名（`codesign`）和公证（`notarytool`），否则用户打开时会被 Gatekeeper 拦截。

---

## 集成到 CI（desktop-build.yml）

在现有 workflow 的对应平台步骤后追加：

```yaml
# Windows
- name: Build NSIS installer
  if: matrix.os == 'windows-latest'
  run: choco install nsis -y && wails build -clean -platform windows/amd64 -nsis

# macOS
- name: Build DMG
  if: matrix.os == 'macos-latest'
  run: |
    brew install create-dmg
    wails build -clean -platform darwin/universal
    create-dmg --volname "Linkit" "build/bin/Linkit.dmg" "build/bin/Linkit.app"
```
