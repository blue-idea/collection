## 1. 产品概述

### 1.1 定位

Linkit 是一款桌面端「智能知识收藏空间」，用于网址、网页资源与灵感素材的收集、整理、发现与再利用。它区别于传统浏览器收藏夹：分类用于长期整理，主题用于跨分类创造组合，AI 帮助理解、连接与整理收藏。

### 1.2 设计目标

- 收藏形成个人知识资产，而非网址列表。
- 分类（Category）稳定组织；主题（Collection）灵活组合。
- AI 协助发现、理解与连接已有收藏。
- 视觉与交互对齐优秀生产力 / 知识管理工具，避免传统文件管理器风格。

### 1.3 运行形态

- 桌面应用：Wails 壳 + React 前端。
- 云能力：Supabase Auth + 用户库数据同步。
- 原型阶段可用 Vite 独立预览 UI；正式交付以 Wails 打包为准。

## 2. 范围定义

### 2.1 MVP 包含（In Scope）

| 领域 | 能力 |
|------|------|
| 账户与存储 | 邮箱登录/注册、本地模式、本地/云双模式对等、切换前确认且不静默覆盖 |
| 书签 | 增删改查、星标、置顶、备注、标签、分类归属、主题归属、拖入 URL 入库 |
| 分类 | 多级树 CRUD；拖拽调整树层级；拖入书签改分类 |
| 主题 | CRUD；拖入书签加入主题；拖出生成新主题组合；AI 按目标创建主题 |
| 标签 | 侧栏筛选、书签打标/去标、采纳 AI 建议标签 |
| 阅读状态 | `unread` / `reading` / `read` / `archived` |
| 浏览视图 | Card、List、Masonry、Timeline（时间流）、Tag Aggregation（标签聚合）、Theme Space（主题空间） |
| 排序与筛选 | 最近访问 / 收藏时间 / 访问次数 / 标题；星标、标签、时间范围筛选 |
| 搜索 | Spotlight 关键词搜索、语义搜索（真实 AI） |
| AI | 入库分析、重新分析、语义搜索、按目标创建主题、去重整理（仅建议需确认）、库内探索推荐、静态知识网络图 |
| 链接健康 | 状态：`ok` / `changed` / `broken`；侧栏含「内容已更新」「失效」入口；扫描可为规则或半自动 |
| 洞察 | 收藏洞察报告（可为规则/半自动） |
| 设置 | 通用、存储、AI（API Base / Model / Key）、外观主题、语言（EN/ZH） |
| 导入导出 | JSON 导出/导入 |
| i18n | 中英切换；默认 English |
| 平台 | Windows + macOS |

### 2.2 非目标 / 后续版本（Out of Scope）

- 真实网页截图作为封面（MVP 可用渐变/占位预览）。
- 外部分享（Share to other apps）。
- 多设备实时协同编辑与冲突三方合并 UI。
- 公网爬取/搜索新 URL 的外链推荐（探索推荐仅限库内关联）。
- 全库力导向可编辑知识大图、社区检测级性能优化。

## 4. 信息架构与界面结构

### 4.1 主窗口（三栏）

```
┌─────────────┬──────────────────────────┬────────────────┐
│ Sidebar     │ Content Area             │ Detail Panel   │

```

- 左侧：资料库、智能入口、分类树、主题、标签、链接健康；可折叠。
- 中间：当前选择下的书签展示、筛选条、视图切换。
- 右侧：选中书签详情、AI 摘要、标签/主题/备注、知识网络入口；可折叠。
- 顶栏：品牌 Linkit、存储徽章（Local / Cloud）、账户、Spotlight、新增、设置。

### 4.2 主要界面文案（默认 English）

| 区域 | 英文 UI（示例） | 中文对照 |

| 智能 | Insights | 收藏洞察 |
| 分类 / 主题 / 标签 | Categories / Collections / Tags | 收藏分类 / 收藏主题 / 标签 |

## 5. 功能说明

### 5.1 账户与启动

| ID | 功能 | 说明 | 验收要点 |
|----|------|------|----------|
| F-AUTH-01 | 登录 | Email + Password | 成功进入主界面；会话可恢复 |
| F-AUTH-02 | 注册 | 创建账户 | 注册成功可登录 |
| F-AUTH-03 | 本地模式 | 无需登录使用 | `storageMode=local`；完整本地能力可用 |
| F-AUTH-04 | 退出登录 | 设置内 Sign Out | 回到登录门；本地数据策略说明书约定为「不自动清空用户导入数据，可恢复种子示例需二次确认」 |
| F-AUTH-05 | 加载态 | Auth 初始化 | 显示加载指示，避免闪屏 |

### 5.2 存储模式

| ID | 功能 | 说明 | 验收要点 |
|----|------|------|----------|
| F-STORE-01 | 本地存储 | 本机持久化资料库与设置 | 重启后数据仍在 |
| F-STORE-02 | 云存储 | 登录后可将整库同步至 Supabase | 徽章显示 Cloud；变更防抖保存 |
| F-STORE-03 | 模式切换 | 本地 ↔ 云 | **切换前确认**；展示将发生的数据影响；**禁止静默覆盖** |
| F-STORE-04 | 导入 / 导出 | JSON | 导出文件可用；导入覆盖前确认 |

切换规则（锁定）：

1. 用户发起切换时弹出确认，说明源与目标、是否覆盖。
2. 默认选项不得自动覆盖对端数据。
3. 用户明确选择「用当前库覆盖目标」或「取消」后方可继续。

## 6. 数据模型（逻辑）

### 6.1 核心实体

**Tag**：`id`, `label`, `color`

**Category**：`id`, `name`, `icon`, `parentId`, `color?`

**Collection（主题）**：`id`, `name`, `emoji`, `color`, `description`, `bookmarkIds[]`

**Bookmark**：

| 字段 | 说明 |
|------|------|
| `title` / `url` / `domain` / `favicon` | 基础信息 |
| `description` / `notes` | 描述与备注 |
| `tags` / `categoryId` / `collectionIds` | 组织关系 |
| `createdAt` / `lastVisitedAt` / `visitCount` | 时间与活跃 |
| `starred` / `pinned` | 标记 |
| `readStatus` | `unread` \| `reading` \| `read` \| `archived`（MVP 新增） |
| `health` | `ok` \| `changed` \| `broken` |
| `aiSummary` / `aiSuggestedTags` | AI 产物 |
| `thumbnail` | 预览键（非真实截图） |

**AppSettings**：`storageMode`, `theme`, `ai`, `locale`（`en` \| `zh`）

**LibraryData**：`bookmarks` + `categories` + `collections` + `tags`

### 6.2 云端

- 表：`user_bookmarks`（`user_id` 唯一，`data` JSONB，`updated_at`）
- RLS：仅本人 SELECT / INSERT / UPDATE / DELETE
- 客户端库变更后防抖写入（原型约 900ms，实现可调）
