
## 开源 vs Mac 商店 vs 独立发布

### 你的项目现状

* 跨平台桌面应用（Windows + macOS + Linux），基于 Wails + Go
* 已有完整 CI/CD，自动构建 DMG / NSIS installer / .deb
* 核心差异化：AI Copilot + Supabase 云同步 + 本地离线模式
* 双语支持（中英文），功能完整度高

---

### 直接结论：**开源 + 独立发布，不建议上 Mac App Store**

**为什么不建议 Mac App Store：**

1. **Wails 应用沙盒问题** ：Mac App Store 强制要求 App Sandbox，Wails 应用使用系统 keyring（

```plaintext
   go-keyring
```

   ）、全局热键（

```plaintext
   golang.design/x/hotkey
```

   ）、系统托盘等，这些在沙盒下要么受限要么需要大量适配工作

1. **审核周期** ：每次更新都要等审核，AI 功能迭代快，会严重拖慢节奏
2. **30% 抽成** ：如果做订阅制，苹果抽 30%，独立开发者很痛
3. **你已有 DMG 构建流程** ，直接分发成本极低

---

### 推荐路径：开源核心 + 商业化增值

**第一阶段：开源建立影响力**

* 在 GitHub 开源，核心功能免费
* 目标：获取 Star、用户反馈、社区贡献
* 你的 README 已经很完整，截图齐全，直接发布即可

**第二阶段：商业化模式选择**

最适合你的是  **Freemium + 云同步订阅** ：

| 功能                    | 免费版  | 付费版 |
| ----------------------- | ------- | ------ |
| 本地存储                | ✅ 无限 | ✅     |
| Supabase 云同步         | ❌      | ✅     |
| AI 功能（自带 API Key） | ✅      | ✅     |
| AI 功能（托管 API）     | 有限额  | 无限   |
| 多设备同步              | ❌      | ✅     |

 **定价参考** ：$4.9/月 或 $39/年，对标 Raindrop.io

**第三阶段：分发渠道**

* GitHub Releases（已有，继续用）
* [Homebrew Cask](https://github.com/Homebrew/homebrew-cask)：

  ```plaintext
  brew install --cask linkit
  ```

  ，macOS 用户最爱
* [Winget](https://github.com/microsoft/winget-pkgs)：Windows 用户
* 自建落地页 + Stripe 收款（比 Paddle 简单）

---

### 关键风险提示

你的商业化核心依赖 Supabase 云同步，需要考虑：

* Supabase 免费额度够不够早期用户
* 用户数据隐私合规（GDPR）
* 如果 Supabase 涨价的备选方案

---

### 一句话总结

 **先开源冷启动，用 GitHub Star 验证需求，再用云同步订阅变现** 。Mac App Store 的沙盒限制和审核成本对你这个技术栈来说得不偿失，独立发布 DMG 你已经有完整流水线了，直接用。
