# Linkit 自动更新方案

Wails v2 本身不内置自动更新，推荐使用 `minio/selfupdate` 配合 GitHub Releases 实现。

## 整体架构

```
GitHub Release (release.yml 已有)
    → 后端 Go: 启动时检查最新 tag，对比当前版本，下载并替换二进制
    → 前端: 收到 Wails 事件后弹出更新提示 UI
```

## 文件改动清单

| 文件 | 操作 | 说明 |
|---|---|---|
| `go.mod` | 添加依赖 | `github.com/minio/selfupdate` |
| `internal/updater/service.go` | 新建 | 检查更新 + 应用更新逻辑 |
| `config/events.go` | 修改 | 添加更新相关事件常量 |
| `main.go` | 修改 | 在 OnStartup 中触发后台检查 |
| `ui/wailsjs/go/...` | 自动生成 | `wails dev` 后自动生成绑定 |
| 前端 | 新建 | 监听事件，弹出更新提示 |

## Step 2: `internal/updater/service.go`

```go
package updater

import (
	"context"
	"encoding/json"
	"fmt"
	"net/http"
	"runtime"
	"strings"

	"github.com/blue-idea/collection/config"
	"github.com/minio/selfupdate"
	wailsruntime "github.com/wailsapp/wails/v2/pkg/runtime"
)

const githubReleasesAPI = "https://api.github.com/repos/blue-idea/collection/releases/latest"

type releaseInfo struct {
	TagName string  `json:"tag_name"`
	Assets  []asset `json:"assets"`
	HTMLURL string  `json:"html_url"`
}

type asset struct {
	Name               string `json:"name"`
	BrowserDownloadURL string `json:"browser_download_url"`
}

type Service struct {
	ctx context.Context
}

func NewService() *Service { return &Service{} }

func (s *Service) SetContext(ctx context.Context) { s.ctx = ctx }

// CheckAndNotify 在后台检查更新，有新版本时通过 Wails 事件通知前端。
func (s *Service) CheckAndNotify() {
	go func() {
		info, err := fetchLatestRelease()
		if err != nil || !isNewer(info.TagName, config.AppVersion) {
			return
		}
		wailsruntime.EventsEmit(s.ctx, config.EventUpdateAvailable, map[string]string{
			"version":    info.TagName,
			"releaseURL": info.HTMLURL,
		})
	}()
}

// ApplyUpdate 下载并替换当前二进制，完成后通知前端重启。
func (s *Service) ApplyUpdate(downloadURL string) error {
	resp, err := http.Get(downloadURL) //nolint:gosec
	if err != nil {
		return fmt.Errorf("download: %w", err)
	}
	defer resp.Body.Close()

	if err := selfupdate.Apply(resp.Body, selfupdate.Options{}); err != nil {
		return fmt.Errorf("apply: %w", err)
	}
	wailsruntime.EventsEmit(s.ctx, config.EventUpdateApplied)
	return nil
}

// GetAssetURL 从 release 资产列表中找到当前平台对应的下载链接。
func (s *Service) GetAssetURL(version string) (string, error) {
	info, err := fetchLatestRelease()
	if err != nil {
		return "", err
	}
	suffix := platformSuffix()
	for _, a := range info.Assets {
		if strings.Contains(a.Name, suffix) {
			return a.BrowserDownloadURL, nil
		}
	}
	return info.HTMLURL, nil // fallback: 打开 release 页面
}

func fetchLatestRelease() (*releaseInfo, error) {
	resp, err := http.Get(githubReleasesAPI) //nolint:gosec
	if err != nil {
		return nil, err
	}
	defer resp.Body.Close()
	var info releaseInfo
	return &info, json.NewDecoder(resp.Body).Decode(&info)
}

// isNewer 比较 "v0.2.0" > "0.1.9"（简单字符串语义版本比较）。
func isNewer(latest, current string) bool {
	latest = strings.TrimPrefix(latest, "v")
	current = strings.TrimPrefix(current, "v")
	return latest > current
}

func platformSuffix() string {
	switch runtime.GOOS {
	case "windows":
		return "windows-amd64"
	case "darwin":
		return ".dmg"
	default:
		return runtime.GOARCH // linux amd64/arm64
	}
}
```

## Step 3: `config/events.go` 添加事件常量

```go
// EventUpdateAvailable 有新版本可用，payload: {version, releaseURL}
EventUpdateAvailable = "linkit:update-available"
// EventUpdateApplied 更新已写入磁盘，需要重启
EventUpdateApplied = "linkit:update-applied"
```

## Step 4: `main.go` 接入 updater

在 `OnStartup` 中加入：

```go
// 在 import 中添加
"github.com/blue-idea/collection/internal/updater"

// 在 main() 中创建服务
updaterService := updater.NewService()

// 在 OnStartup 回调中
updaterService.SetContext(ctx)
updaterService.CheckAndNotify() // 后台异步检查

// 在 Bind 列表中添加
updaterService,
```

## Step 5: 前端监听事件（React）

在应用根组件（如 `App.tsx`）中：

```tsx
import { useEffect, useState } from 'react';
import { EventsOn } from '../wailsjs/runtime/runtime';
import { ApplyUpdate, GetAssetURL } from '../wailsjs/go/updater/Service';

export function UpdateNotifier() {
  const [update, setUpdate] = useState<{ version: string; releaseURL: string } | null>(null);

  useEffect(() => {
    return EventsOn('linkit:update-available', (payload) => setUpdate(payload));
  }, []);

  useEffect(() => {
    return EventsOn('linkit:update-applied', () => {
      alert('Update applied! Please restart Linkit.');
    });
  }, []);

  if (!update) return null;

  const handleUpdate = async () => {
    const url = await GetAssetURL(update.version);
    // macOS DMG 需要手动安装，直接打开 release 页面
    if (url.endsWith('.dmg') || url.startsWith('https://github.com')) {
      window.open(url, '_blank');
    } else {
      await ApplyUpdate(url);
    }
    setUpdate(null);
  };

  return (
    <div className="fixed bottom-4 right-4 bg-surface border border-border rounded-lg p-4 shadow-lg z-50">
      <p className="text-sm font-medium">New version {update.version} available</p>
      <div className="flex gap-2 mt-2">
        <button onClick={handleUpdate} className="btn-primary text-xs">Update</button>
        <button onClick={() => setUpdate(null)} className="btn-ghost text-xs">Later</button>
      </div>
    </div>
  );
}
```

## 平台注意事项

**Windows**：运行中的 `.exe` 无法直接替换自身，`selfupdate` 库用重命名技巧绕过，但需要管理员权限或安装目录可写。更稳妥的方案是直接打开 release 页面让用户手动下载 NSIS installer。

**macOS**：替换 `.app` 内的二进制会破坏代码签名，建议始终引导用户下载新 DMG。

**Linux**：`selfupdate` 直接替换二进制可正常工作。

## 最简方案（仅通知，不自动替换）

如果不想处理平台差异，所有平台只做"检查 + 通知 + 打开浏览器"：

```go
func (s *Service) CheckAndNotify() {
    go func() {
        info, err := fetchLatestRelease()
        if err != nil || !isNewer(info.TagName, config.AppVersion) {
            return
        }
        wailsruntime.EventsEmit(s.ctx, config.EventUpdateAvailable, map[string]string{
            "version":    info.TagName,
            "releaseURL": info.HTMLURL,
        })
    }()
}
```

前端收到事件后用 Wails runtime 的 `BrowserOpenURL` 打开 GitHub release 页面，无需 `selfupdate` 依赖，最为安全稳定。
